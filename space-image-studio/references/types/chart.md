# 图表（按子类型自适应比例）

| # | 类型 | 英文 | 比例 | 适用场景 | 推荐风格 |
|---|------|------|------|----------|----------|

| 2 | 架构图 | Architecture | 16:9 | 系统架构、模块关系 | blueprint |
| 3 | ER 图 | ER Diagram | 16:9 | 数据模型、表关系 | blueprint |
| 4 | 思维导图 | Mind Map | 4:3 | 头脑风暴、知识结构 | sketch-notes |

## 通用必填字段

- ① 图表类型（从 12 种选）
- ② 节点 / 实体清单
- ③ 节点之间的关系（顺序 / 层级 / 连接 / 数值）
- ④ 强调项 / 焦点（可选）

## 各类型特殊规则

### 流程图
- 起止节点用圆角矩形，决策点用菱形
- 主流程粗线，分支细线
- 节点 ≤ 10 个，超过则拆分

### 架构图
- 分层清晰：用户层 / 应用层 / 服务层 / 数据层
- 同层模块对齐
- 数据流向用箭头标注

### ER 图
- 实体用矩形，关系用菱形
- 标注基数（1:1 / 1:N / N:M）
- 字段列在实体框内

### 思维导图
- 中心主题居中，向外辐射
- 一级分支不超过 6 条
- 颜色按分支区分

### 用户旅程图
- 横向时间线（认知 → 兴趣 → 决策 → 使用 → 推荐）
- 上行：触点 / 行为
- 下行：情绪曲线 / 痛点机会

### 商业模式画布
- 严格 9 宫格：客户细分 / 价值主张 / 渠道 / 客户关系 / 收入来源 / 核心资源 / 关键活动 / 重要伙伴 / 成本结构
- 每格 ≤ 30 字

### SWOT
- 严格 2×2：S / W / O / T
- 每象限 3–5 条
- 用色区分：S 蓝 / W 红 / O 绿 / T 橙

### 竞品分析图
- 多维雷达图 或 二维定位图
- 突出自身位置
- 维度标签清晰

### 产品路线图
- 横向时间轴（Q1 / Q2 / Q3 / Q4）
- 纵向分泳道（功能 / 技术 / 体验）
- 里程碑用图标标注

### 组织架构图
- 上下层级清晰
- 同级对齐
- 部门 / 职位分两行

### 热力图
- 矩阵布局（行 × 列）
- 颜色梯度：深色高、浅色低
- 数值可标注在格内
- 必须有图例

### 统计图
- 柱状：横/纵清晰，数值标注
- 折线：颜色不超过 4 条
- 饼图：扇区不超过 6 个，按大小排序

## 禁止

- 节点 > 12 个一律拆图
- 不出现网格背景（除 heatmap）
- 不用花哨阴影
- 文字必须能完整显示，禁止溢出
